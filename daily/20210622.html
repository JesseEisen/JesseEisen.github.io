这是我尝试开始使用 TODO 进行每天日常记录的实验版本，主要是参照 Roam Research 的一些做法，只是方式比较原始一些，但是也能满足我的使用。

  • 模型管理服务 2.0 细节优化

    • 使用 Time Boxing 的方式，尝试对 2.0 版本的上线进行一个系统调试和测试
    • Steps:
      • 调试除 Hiai C30/C32 平台外，其他平台的转模型和测试功能
        • 10:20 看到线上测试总是报错，花费 10mins 协助定位问题，接下来继续调试 Python 的测试工程
        • 11:55 增加了 Pascal 的转模型，测试没问题。中间被其他的任务打断，主要是去协助定位模型包和测试的问题。不过插入一个新需求，需要优先解决。
        • 17:13 在增加了显卡的选择之后，Pascal 的转模型和测试都可以了。其他依赖 ezi 之前那套转模型测试的 engine 需要改一下里面的内容。
      • 增加性能测试时，用户自定义选择测试卡。
        • 13:16 花费 10mins 进行需求梳理，准备直接在 2.0 上实现这个功能。
        • 16:04 页面修改和下发控制完成，功能完成，耗时接近 3 小时，中间大部分时间在调 Python 脚本支持 shell 的一些命令，效率值一般。

  • 支持 GTD 的日常记录

    • 17:02 当前页面就是支持了之后的效果，感觉还可以，修改了一下字体大小和颜色，有别于笔记和博客。此外增加了两个组件用于标记任务完成和待办。
